 # 💻 Windows 本地开发环境配置指南 (基于 Docker)

## 阶段一：基础设施搭建 (Docker 环境)

在 Windows 上运行 Docker，我们需要借助微软官方的 WSL 2（Windows Subsystem for Linux），这能让 Docker 运行得极其流畅。

1. 开启 WSL 2

以管理员身份打开 PowerShell 或命令提示符 (CMD)。

输入以下命令并回车，等待安装完成：

```powershell
wsl --install
```

安装完成后，重启你的电脑。

2. 安装 Docker Desktop

前往 Docker 官网下载 Docker Desktop for Windows。

运行安装程序，安装过程中确保勾选了 "Use WSL 2 instead of Hyper-V"（通常默认会勾选）。

安装完成后打开 Docker Desktop，如果左下角显示绿色的 "Engine running"，说明 Docker 已成功启动。

## 阶段二：一键部署中间件 (MySQL & Redis)

有了 Docker，我们不再需要去官网下载繁琐的 MySQL 和 Redis 安装包，直接通过两行命令即可在容器中启动它们。

打开 PowerShell 或 CMD。

启动 MySQL 8 容器：

```bash
docker run -d --name mysql-local -p 3306:3306 -e MYSQL_ROOT_PASSWORD=root mysql:8.0
```

说明：这行命令会拉取 MySQL 8.0 镜像，将密码设置为 root，并映射到你本地的 3306 端口。

3. 启动 Redis 容器：

```bash
docker run -d --name redis-local -p 6379:6379 redis
```

说明：这行命令会拉取最新的 Redis 镜像，并映射到你本地的 6379 端口。

💡 测试连接：

建议下载一个免费的数据库可视化工具 DBeaver 或 Navicat。

- 连接 MySQL：主机填 localhost，端口 3306，账号 root，密码 root。
- 连接 Redis：主机填 localhost，端口 6379，无密码。

## 阶段三：核心开发环境配置 (Java & Maven)

1. 安装 JDK 17

前往 Oracle 官网 或 Adoptium 下载 JDK 17 的 Windows x64 Installer (.exe 格式)。

按照提示无脑下一步安装。

配置环境变量：

右键“此电脑” -> 属性 -> 高级系统设置 -> 环境变量。

在“系统变量”中新建：变量名 JAVA_HOME，变量值填入你的 JDK 安装路径（如 C:\\Program Files\\Java\\jdk-17）。

找到 Path 变量，编辑并新建一条：%JAVA_HOME%\\bin。

验证： 打开 CMD，输入 `java -version`，如果输出 Java 17 的版本信息即为成功。

2. 安装与配置 Maven (极度重要：配置阿里云镜像)

下载 Maven 3.8+ 版本 的 .zip 压缩包，解压到你的常用软件目录（如 D:\\maven）。

配置环境变量： 同样新建 MAVEN_HOME，指向解压目录，并将 %MAVEN_HOME%\\bin 加入 Path。

配置阿里云镜像（防止下载依赖卡死）：

打开 Maven 解压目录下的 `conf/settings.xml` 文件。

找到 `<mirrors>` 标签，在其中插入以下代码：

```xml
<mirror>
  <id>aliyunmaven</id>
  <mirrorOf>*</mirrorOf>
  <name>阿里云公共仓库</name>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

## 阶段四：配置智能 IDE (Cursor)

1. 下载并安装 **Cursor**。
2. 打开 Cursor，按快捷键 `Ctrl + Shift + X` 打开扩展（Extensions）面板。
3. 搜索并安装以下必备插件：
   - **Extension Pack for Java** (微软官方提供，包含基础语法提示等)
   - **Spring Boot Extension Pack** (包含 Spring 相关的支持)
   - **Maven for Java** (方便执行 Maven 命令)
4. 将我们上一轮写好的 `.cursorrules` 文件，放入你即将创建的 Spring Boot 项目根目录中。

## 阶段五：准备大模型 API Key

为了后续开发“多模态题库导入”功能，你需要提前准备好大模型的 API 密钥。

1. 访问 **月之暗面 (Kimi)** 开放平台 (platform.moonshot.cn) 或 **智谱 AI** 开放平台 (open.bigmodel.cn)。
2. 注册账号并完成实名认证。
3. 在控制台的“API Key 管理”中，生成并复制你的专属 API Key（通常以 `sk-` 开头）。
4. **妥善保存这串密钥**，在后续配置 Spring Boot 的 `application.yml` 文件时，我们会将其填入配置文件中。
