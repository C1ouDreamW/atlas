太棒了！我们在本地经历了从建库、写核心业务、对接 AI 到 Redis 高并发优化的全流程，甚至连 JMeter 压测都跑通了。现在，终于到了把这辆“跑车”开上主干道（云服务器）的时候了。

在企业级部署中，为了保证环境的一致性和运维的便捷性，我们依然会重度依赖 Docker。特别是对于 MySQL 和 Redis 这种中间件，用 Docker 部署是业界的绝对主流。

以下为你量身定制的《基于 Ubuntu 的智能在线刷题平台生产环境部署指南》。

# 🚀 Ubuntu 服务器生产环境配置与部署指南

**环境假设**：你拥有一台全新安装了 Ubuntu 20.04 LTS 或 22.04 LTS 的云服务器（如阿里云、腾讯云），并且已经通过 SSH 登录。

## 阶段一：基础安全与运维工具准备

刚拿到服务器，先别急着装环境，先做一波系统更新，并安装一些必要的运维工具。

```bash
# 1. 更新系统软件包列表
sudo apt update && sudo apt upgrade -y

# 2. 安装基础工具（网络排查、进程查看等）
sudo apt install -y curl wget git vim net-tools htop
```

## 阶段二：一键部署核心中间件 (Docker & Docker Compose)

在线上环境，我们不推荐像本地那样用长长的 docker run 命令，而是使用 Docker Compose 来编排，这样即使服务器重启，数据库和缓存也能自动拉起。

### 1. 安装 Docker 与 Docker Compose

```bash
# 安装 Docker
sudo apt install -y docker.io

# 启动并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo apt install -y docker-compose
```

### 2. 编排 MySQL 与 Redis

在服务器的用户主目录（如 `/home/ubuntu`）下创建一个工作目录：

```bash
mkdir -p ~/quiz-env && cd ~/quiz-env
vim docker-compose.yml
```

按 `i` 键进入编辑模式，将以下内容粘贴进去（注意：这里我做了一个架构师级别的安全优化：将端口映射写成了 `127.0.0.1:3306`，这样数据库就不会暴露在公网上被黑客爆破）：

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    container_name: quiz-mysql-prod
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: YourStrongPasswordHere # ⚠️ 务必改成极复杂的密码
    ports:
      - "127.0.0.1:3306:3306" # 仅允许服务器本地连接
    volumes:
      - ./mysql-data:/var/lib/mysql # 数据持久化，防止容器删除后数据丢失

  redis:
    image: redis:latest
    container_name: quiz-redis-prod
    restart: always
    ports:
      - "127.0.0.1:6379:6379"
```

按 `Esc`，输入 `:wq` 保存退出。然后一键启动：

```bash
sudo docker-compose up -d
```

启动后，别忘了通过命令行或临时开放端口，把你本地导出的那个 `smart_quiz_db` 的 SQL 结构导入到服务器的 MySQL 中！

## 阶段三：配置 Java 运行环境

Spring Boot 3 最大的好处是它内嵌了 Tomcat，打出来的是一个包含了所有依赖的 Fat JAR 包。所以服务器上不需要装 Tomcat，只需要装 Java 17 即可。

```bash
# 安装 OpenJDK 17（Headless 版本更轻量，适合没有图形界面的服务器）
sudo apt install -y openjdk-17-jre-headless

# 验证安装
java -version
```

## 阶段四：打包与上传项目代码

### 1. 本地打包 (在你的电脑上执行)

打开 Cursor 的终端，在项目根目录下执行 Maven 打包命令。这会跳过测试，直接生成可执行文件。

```bash
mvn clean package -DskipTests
```

执行完毕后，在项目的 `target` 目录下会生成一个类似 `quiz-backend-0.0.1-SNAPSHOT.jar` 的文件。

### 2. 上传到服务器

如果你用的是 FinalShell、Xshell 或 Tabby，直接把这个 JAR 包拖拽到服务器的 `~/quiz-env` 目录下即可。
或者在本地终端使用 `scp` 命令：

```bash
scp target/quiz-backend-0.0.1-SNAPSHOT.jar ubuntu@你的服务器公网IP:~/quiz-env/
```

## 阶段五：配置生产环境并启动应用

线上环境的数据库密码、大模型 API Key 肯定和本地不一样。我们不需要重新打包，只需在 JAR 包同级目录下建一个配置文件覆盖默认配置即可。

```bash
cd ~/quiz-env
vim application-prod.yml
```

填入线上配置（这里填上真实的密码和 API Key）：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/smart_quiz_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: YourStrongPasswordHere
  data:
    redis:
      host: 127.0.0.1
      port: 6379

ai:
  llm:
    api-key: sk-你的真实线上APIKEY
```

🔥 激动人心的启动时刻 (使用 nohup 后台运行)：

```bash
# 指定使用 application-prod.yml 配置，并将日志输出到 app.log 中
nohup java -jar quiz-backend-0.0.1-SNAPSHOT.jar --spring.config.location=./application-prod.yml > app.log 2>&1 &
```

敲完回车后，输入 `tail -f app.log` 查看实时日志。如果看到熟悉的 `Started QuizBackendApplication in X.XX seconds`，恭喜你，后端服务已在云端成功跑起！

## 阶段六：配置 Nginx 代理 (简历级规范)

让用户在浏览器里输入 `http://你的IP:8080` 访问是非常不专业的。正规做法是用 Nginx 监听 80 端口，并转发给 Java。

```bash
sudo apt install -y nginx
sudo vim /etc/nginx/sites-available/default
```

将里面的 `location /` 块替换为：

```nginx
location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

重启 Nginx：

```bash
sudo systemctl restart nginx
```
